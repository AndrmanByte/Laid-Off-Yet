# SECURITY

本仓库是职业支持类 Agent Skill 项目，不是漏洞利用项目。  
重点是内容安全、隐私保护和提示词安全。

## 安全边界

- 不提供违法、破坏、报复类建议
- 不提供确定法律结论
- 不提供医疗或心理诊断
- 不接受包含敏感真实数据的示例

## 严禁上传的敏感材料

请不要在 Issue、PR、示例或任何文件中上传以下内容：

- 真实裁员协议原文（含公司与个人信息）
- 真实工资单、社保流水、银行信息
- 身份证、手机号、住址、邮箱等可识别隐私
- 公司机密文档、内部代码、客户数据
- 可定位真实个人或企业的敏感信息

建议做法：
- 用脱敏后的虚拟样例替代真实材料
- 用占位符替代姓名、公司、金额、日期

## Prompt 注入与内容安全风险

如发现以下问题，请尽快报告：

- Prompt 注入导致输出越界（例如生成违法建议）
- 话术模板引导用户进行破坏性行为
- 示例中泄露隐私或可识别真实对象
- 分流逻辑错误导致高风险误导

## 如何报告安全问题

1. 新建 Issue，标题建议：`[security] 问题简述`
2. 说明：
- 触发文件路径
- 问题类型（注入/隐私/误导/越界）
- 复现步骤（尽量最小化）
- 预期修复方向

请不要在公开 issue 中粘贴真实敏感数据。

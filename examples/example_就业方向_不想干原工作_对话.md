# 示例：被裁后不想回原岗位，还能做什么？

## 用户输入

「我被裁了，不想回原岗位了，还能做什么？」

## Skill 输出要点（结构化 · 对齐 data/）

1. **先稳住**：不想回原岗可讨论；先区分「厌恶环境」与「厌恶能力类型」，避免冲动定型。
2. **当前局面判断**：**就业方向咨询分支**；先按 `data/intake_questions.json` 补齐城市、储蓄月数、家庭压力、能否接受销售/波动等。
3. **现在最重要的事**：用 `data/recommendation_rules.json` 做约束过滤，再从 `data/role_profiles.json` 选 **3～5 个** `id`（如 `operations_content`、`admin_assistant`、`cs_customer_service` 等视约束而定），**不直接给唯一答案**。
4. **今天/本周行动清单**：本地招聘软件检索目标 `id` 对应岗位 JD；记录共性要求；完成背景问答缺口列表。
5. **可以说的话（家人）**：「我在用结构化清单筛方向，有试水计划，不是拍脑袋。」
6. **风险提醒**：跨岗常伴随起薪/职级变化；收入以实时 JD 为准。
7. **时间线**：7 天补信息+初筛；30 天单一方向试水（作品/投递/访谈）。
8. **行动收束**：用户自选主试方向；每条输出须含 **difficulty、recommendation_score、技能差距、trial_plan_30d**（字段来自 JSON）。

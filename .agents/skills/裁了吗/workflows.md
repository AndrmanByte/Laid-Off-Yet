# 工作流

## 核心执行顺序（固定）

所有场景都按以下顺序执行，不跳步：

1. 情绪稳定
2. 局面判断
3. 决策分流
4. 信息与策略
5. 行动清单
6. 风险提醒
7. 时间线
8. 行动收束

## 输出约束

- 大多数场景按完整 8 段输出。
- 用户只问单点问题时，也先给前 3 段，再给对应行动和话术。

---

## Workflow 1：刚被裁员

1. 情绪稳定：先止住冲动表达，提示“今晚不做不可逆决定”。
2. 局面判断：确认是否已收到正式通知、是否有签字时限、是否涉及离场交接。
3. 决策分流：进入“裁员流程分支”，判断是否同时命中“情绪稳定分支”。
4. 信息与策略：优先拿书面信息、确认补偿口径、确认关键日期。
5. 行动清单：今天做证据保存+HR确认；本周做条款核对+现金流盘点+求职启动。
6. 风险提醒：不在信息不完整时签字，不在公开平台情绪化发言。
7. 时间线：7天完成协商与求职起步；30天稳定执行；90天路径收敛。
8. 行动收束：总结成“今晚三件事”和“明天第一件事”。

## Workflow 2：HR 谈判

1. 情绪稳定：先把“被施压感”降到可沟通状态。
2. 局面判断：确认谈判阶段（未签/已签）、是否催签、是否谈补偿或附加义务。
3. 决策分流：进入“裁员流程分支-谈补偿/签字子分支”。
4. 信息与策略：先争取时间，再要书面，再核条款（基数、月份、税费、发放、附加条款）。
5. 行动清单：今天发延期申请与问题清单；本周完成会后书面纪要和条款复核。
6. 风险提醒：口头承诺不等于生效条款；复杂争议需专业咨询。
7. 时间线：7天完成关键信息确认；30天完成过渡安排；90天稳定新路径。
8. 行动收束：给出可直接发送的 HR 文本 + 下一次跟进时间点。

## Workflow 3：情绪崩溃

1. 情绪稳定：先做 3-10 分钟稳定动作（呼吸、补水、停止冲动行为）。
2. 局面判断：判断是否影响睡眠/进食/基本功能，是否有高风险信号。
3. 决策分流：进入“情绪稳定分支”，必要时叠加“行动计划分支”。
4. 信息与策略：把问题拆成“今晚、明天、本周”三级，不做终局判断。
5. 行动清单：今天只做 1-2 个现实动作；本周每天 1-3 个关键任务。
6. 风险提醒：持续失眠、明显功能受损或极端表达时，优先专业支持。
7. 时间线：7天恢复执行力；30天恢复节律；90天完成稳定转轨。
8. 行动收束：明确“今天到此为止”的收口动作。

## Workflow 4：未来方向选择

1. 情绪稳定：先降低“身份焦虑”，避免情绪替代决策。
2. 局面判断：确认当前约束（现金流、家庭压力、地区机会、精力状态）。
3. 决策分流：进入“方向决策分支”。
4. 信息与策略：给主路径+备路径，设置 14 天验证指标（反馈率/面试数/可持续性）。
5. 行动清单：今天列路径对比；本周执行验证动作并记录数据。
6. 风险提醒：只靠直觉切换方向，成本高且可重复犯错。
7. 时间线：7天起跑验证；30天完成方向收敛；90天形成稳定节奏。
8. 行动收束：给出“继续/切换”的明确触发条件。

## Workflow 5：转行决策

1. 情绪稳定：先确认“转行冲动”是否来自短期挫败。
2. 局面判断：确认目标方向是否明确、是否有可迁移能力和试错窗口。
3. 决策分流：进入“转行分析分支”。
4. 信息与策略：先低风险试错（作品/访谈/小规模投递），再决定是否加码。
5. 行动清单：今天确定目标岗位；本周产出最小作品并完成行业访谈。
6. 风险提醒：冲动裸辞转行、无作品直接转岗、只看热度不看岗位密度。
7. 时间线：7天完成目标收敛；30天形成可验证证据；90天做去留决策。
8. 行动收束：给出回撤条件和兜底方案。

## Workflow 6：单干 / 副业

1. 情绪稳定：先从“逃离心态”切回“验证心态”。
2. 局面判断：确认是副业试水还是全职单干，确认现金流窗口。
3. 决策分流：进入“单干副业分支”。
4. 信息与策略：定义一句话服务，明确最低收入阈值与继续条件。
5. 行动清单：今天完成服务定义；本周完成触达、沟通、试点准备。
6. 风险提醒：无付费信号继续加码、边界不清导致交付失控。
7. 时间线：7天启动触达；30天判断可行性；90天决定主副路径。
8. 行动收束：明确“继续/暂停/降级为副路径”的判定点。

## Workflow 7：行动计划

1. 情绪稳定：先确认当下可执行精力，避免计划过载。
2. 局面判断：识别用户当前阶段（0-7天/8-30天/31-90天）。
3. 决策分流：进入“行动计划分支”，并挂接主问题分支（裁员/情绪/方向/转行/单干）。
4. 信息与策略：把目标拆成“今天、本周、本月、三个月”。
5. 行动清单：给可执行任务，今天 3-5 条、本周 5-8 条。
6. 风险提醒：计划过大、任务过多、无复盘节点会导致放弃。
7. 时间线：7天看执行率；30天看结果信号；90天看路径稳定性。
8. 行动收束：给“下一次复盘时间 + 复盘问题清单”。

## Workflow 8：就业方向咨询 / 职业方向推荐

**适用**：用户问「适合做什么」「我还能干什么」「能干什么」「不想回原行业/原岗位」「我想转行」「转行 XX 可以吗」「现实找什么工作」「我想做副业」「养活自己」「小生意」等，需要**结构化数据支撑**的方向分析时。

**数据与文档**：优先使用仓库根目录 `data/role_profiles.json`、`career_paths.json`、`skill_mappings.json`、`recommendation_rules.json`、`intake_questions.json`；人类可读模板见 `resources/就业方向*.md`、`岗位画像库.md`、`转岗技能映射.md`。

1. **情绪稳定**：先接住焦虑/羞耻，避免在情绪峰值做「终身定型」；声明 Skill **不替用户做决定**、不保证结果。  
2. **了解背景**：按 `data/intake_questions.json`（及 `就业方向推荐流程.md`）收集维度；信息不足则 **先提问再推荐**。  
3. **判断限制条件**：标出最硬约束；用 `data/recommendation_rules.json` + `就业方向评估方法.md` 说明筛选逻辑。  
4. **推荐方向**：从 `data/role_profiles.json` 筛 **3～5** 个 `id`；开放选题与已点名某方向的路径见 `就业方向推荐流程.md`。  
5. **岗位要求说明**：字段来自 `role_profiles`：`job_requirements`、`core_skills`。  
6. **技能差距**：`data/skill_mappings.json` 中匹配 `from_background` → `target_role_id`，并合并 `skills_to_build`。  
7. **上手难度 / 推荐指数**：使用 JSON 中 `difficulty`、`recommendation_score`，并声明非职业排名。  
8. **试水建议**：使用 `trial_plan_30d`；整体设 7 天可验证指标。  
9. **行动清单**：今天/本周：补信息、投递/访谈/试点；可叠加 `找工作-checklist.md` 或 `单干试水-checklist.md`。  
10. **风险提醒**：收入区间以本地 JD 为准；自媒体等幸存者偏差；小生意亏损与合规。  
11. **时间线**：7 天/30 天/90 天复盘节点。  
12. **行动收束**：请用户自选「主试 1 + 备选 1～2」，Skill 只提供利弊与试水路径。

**输出结构**：在默认 **8 段结构**下展开；风格：**中文、冷静、现实、结构化**，给出难度/指数/差距/试水，**不画大饼**。

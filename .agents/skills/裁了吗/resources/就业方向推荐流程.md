# 就业方向推荐流程（内容模板）

## 说明

- **问题清单**：`data/intake_questions.json`（分 section，含 `prompt` / `why` / `maps_to_rules`）。  
- **规则**：`data/recommendation_rules.json`。  
- **画像**：`data/role_profiles.json` + `data/career_paths.json`。

## 流程（不直接给唯一答案）

### 1. 先收集背景

按 `intake_questions.json` 顺序 **分批提问**（不必一轮问完）。  
若用户拒绝透露部分信息：声明 **推荐置信度下降**，并给更保守的 3 个方向 + 待补信息列表。

### 2. 将回答映射到规则维度

年龄阶段、城市、原行业/岗位、储蓄月数（→ 收入恢复紧迫度）、家庭压力、是否愿学习/销售/体力/不稳定收入等，对应 `recommendation_rules.json` 的 `dimensions`。

### 3. 筛选方向（3～5 个）

- 用 `rules` 中的 `boost` / `penalize` 做启发式排序（见 `selection_policy`）。  
- 从 `role_profiles.json` 取完整字段；总览层可并列读 `career_paths.json`。  
- 用户**已点名某方向**（如运营）：对该 `target_role_id` 做可行性分析，并从其余 id 中选 1～2 个兜底。

### 4. 结构化输出（每个方向）

- 岗位要求：`job_requirements` + `core_skills`  
- 技能差距：结合 `skill_mappings.json` 与用户背景 + `skills_to_build`  
- 上手难度：`difficulty`  
- 推荐指数：`recommendation_score`  
- 试水：`trial_plan_30d`  

### 5. 收束

请用户自选 **主试 1 + 备选 1～2**；Skill 只提供利弊与 **7 天可验证指标**（投递数、访谈数、试点收入等）。

## 与 8 段输出的关系

默认仍遵循全仓「先稳住 → … → 行动收束」；本流程的主体落在 **局面判断、行动清单、风险提醒、时间线**。

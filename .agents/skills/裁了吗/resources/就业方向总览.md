# 就业方向总览

## 模块定位

回答「我适合做什么」「我还能干什么」「我想转行做 XX 可以吗」「不想回原行业」「我想做副业/养活自己」等，**基于结构化数据做方向分析**，避免泛泛建议。  
**不替用户做决定**：先收背景 → 按规则筛选 → 输出 3～5 个方向及画像字段（要求、差距、难度、指数、试水）。

## 何时进入「就业方向咨询」

在 `decision-tree.md` 中命中 **就业方向咨询分支** 时启用，典型表达包括：

- 我适合做什么 / 我能干什么 / 我还能干什么  
- 我想转行（含「转行做 XX 可以吗」）  
- 我不想回原行业 / 不想回原岗位  
- 我想做副业 / 自由职业 / 能做什么养活自己  
- 现在找什么工作比较现实（常叠学历/年龄/空窗）

若与「裁员签字」「情绪崩溃」同时出现，仍优先情绪与法定流程，再进入本模块。

## 数据源（仓库根目录，权威字段）

画像覆盖 **受雇白领/职能、平台接单、服务业一线、仓储物流、小微门店与副业** 等，见 `role_profiles.json` 中 `category`（含 `platform`、`service`、`employment`、`flexible`）。

| 文件 | 用途 |
|------|------|
| `data/role_profiles.json` | 各方向完整画像（含 `common_entry_barriers` 常见进入门槛） |
| `data/career_paths.json` | 方向总览（类型、描述、风险、难度、指数） |
| `data/skill_mappings.json` | 原背景 → 目标方向：可迁移技能、缺口、转岗说明 |
| `data/recommendation_rules.json` | 基于约束的加权/降权启发式规则 |
| `data/intake_questions.json` | 优先询问用户的问题清单与映射维度 |

Skill 输出时应优先与上述 JSON **字段对齐**；若 Agent 无法读文件，可依赖本目录 `岗位画像库.md` 的字段模板作 fallback。

## 读取顺序建议

1. `就业方向推荐流程.md`：如何收背景、筛选、输出  
2. `就业方向评估方法.md`：难度/指数含义与声明  
3. `data/intake_questions.json`：缺信息时问什么  
4. `data/recommendation_rules.json` + `data/role_profiles.json`：生成 3～5 个候选方向  
5. `data/skill_mappings.json`：个性化技能差距表述  
6. `转岗技能映射.md`：人类可读摘要（与 JSON 一致）

## 输出原则

- 中文，冷静，现实；**不夸大**前景与收入。  
- 收入、易就业等表述须声明**地域与个体差异**，数据仅为启发式。  
- 每个推荐方向须包含：**岗位要求摘要、技能差距、上手难度、推荐指数、30 天试水**（字段来自 `role_profiles.json`）。
